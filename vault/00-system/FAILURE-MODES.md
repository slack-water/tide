## over-optimization spiral

**the pattern:** building and refining the system instead of using it. spending time on configuration, structure, and tooling rather than the work the system is meant to support.

**early signals:**

- more time spent on the system than on what the system is for
- another folder, template, or script before writing a single substantive note
- the tooling feels exciting; the actual work feels flat

**the cost:** the system becomes the product. the real work never gets done.

**the redirect:** close the config. open the inbox. write one real thing before touching structure again.

**accountability:** if you've made more than one structural change this week without a corresponding substantive note, you're in this pattern.
