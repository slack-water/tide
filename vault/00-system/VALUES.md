# Values

*personal values definition. upstream of everything — folder weights, project prioritization, and the weekly review queue all derive from this. write it before you write anything else.*

*this is a discovery document, not a performance. the goal is to name what you actually care about, not what sounds right.*

---

## how to write this

values work best when they're specific enough to make decisions with. "family" is not a value. "being present with the people i love, which sometimes means closing the laptop" is a value.

a useful structure for each value:

```markdown
## [value name]

[one or two sentences: what this means to you specifically, and why it matters]

**what it looks like in practice:** [concrete behavior — what you do, not what you believe]
**what violates it:** [the thing you're actually protecting against]
```

---

## example

## depth over breadth

i'd rather understand one thing well than have a surface familiarity with ten things. this shows up in how i work, what i read, and how i build relationships — i'm slow to start but hard to move once i'm in.

**what it looks like in practice:** i finish books. i stay in conversations longer than is comfortable. i resist the urge to pick up new tools before i've mastered the current ones.

**what violates it:** context-switching as a form of avoidance. filling a calendar with shallow commitments. treating familiarity as expertise.
